# mintalib Design Synthesis

A portable summary of mintalib's design knowledge, intended to inform related projects (e.g. barcalc).
mintalib is a Cython-based technical analysis library for Python. barcalc is a similar package using numba.

---

## Architecture

The library is organized in layers:

```
src/mintalib/cython/<name>.pxi   one file per indicator, implements calc_<name>
src/mintalib/cython/_all_core.pxi  includes all .pxi files into core.pyx
src/mintalib/core.pyx            Cython extension, exposes all calc_* functions
src/mintalib/functions.py        thin wrappers, primary stable interface (generated)
src/mintalib/indicators.py       composable indicator objects (generated)
src/mintalib/expressions.py      polars expression factories (generated)
src/mintalib/model/              wrap_function, wrap_indicator, wrap_expression decorators
src/mintalib/pandas.py           experimental ts accessor for pandas (opt-in)
src/mintalib/polars.py           experimental ts accessor for polars (opt-in)
```

`functions.py`, `indicators.py`, and `expressions.py` are **generated** by notebooks in `scripts/` тАФ never edited directly.

---

## Naming Conventions

| Layer | Convention | Example |
|---|---|---|
| Core (Cython) | `calc_` prefix, lower case | `calc_sma`, `calc_macd` |
| Functions | lower case | `sma`, `macd` |
| Indicators | upper case | `SMA`, `MACD` |
| Expressions | upper case | `SMA`, `MACD` |

---

## Input Parameters

Two input conventions, determined by the first parameter name:

- `series` тАФ a single pandas/polars Series or numpy array
- `prices` тАФ a DataFrame with columns `open`, `high`, `low`, `close`, `volume` (all lower case)

Price-based indicators (`adx`, `atr`, `bbands`, etc.) require a `prices` DataFrame.
Series-based indicators (`sma`, `ema`, `rsi`, etc.) accept a single series.

---

## Multi-Output Indicators

Indicators with multiple outputs (e.g. `macd`, `bbands`, `dmi`) return:

- **core / functions** тАФ named tuples (e.g. `macd_result(macd, macdsignal, macdhist)`)
- **expressions** тАФ polars struct expression, unpack with `.unnest()`
- **polars accessor** тАФ polars struct

---

## The Four Stable Interfaces

### 1. `mintalib.core`
Raw Cython functions. Named `calc_<name>`. Accept numpy arrays or pandas/polars series/dataframes.
The `wrap` parameter controls whether the result is wrapped to match the input type.

### 2. `mintalib.functions`
Thin Python wrappers around core. Named lower case (`sma`, `ema`).
Handle input/output type conversion automatically.
Some names (`abs`, `min`, `max`, `sum`) shadow Python builtins тАФ always import with alias:
```python
import mintalib.functions as ta
```

### 3. `mintalib.indicators`
Callable objects that bind calculation + parameters. Named upper case (`SMA`, `EMA`).
Support `@` operator for chaining:
```python
SMA(20) @ prices          # apply to data
ROC(1) @ EMA(20)          # chain indicators
SMA(50)(prices)           # alternative call syntax
```
Work with pandas and numpy.

### 4. `mintalib.expressions`
Polars expression factories. Named upper case (`SMA`, `EMA`).
The `src` keyword parameter overrides the default input column (default: `CLOSE = pl.col('close')`).
Price-based indicators have `src=None` (not applicable).
```python
prices.select(SMA(20), EMA(20))
prices.with_columns(MACD().unnest())
```

---

## Experimental Interfaces

### `mintalib.pandas` тАФ `ts` accessor
Registers a `ts` accessor on pandas DataFrame and Series.
Import the module to activate тАФ no explicit registration call needed.
```python
import mintalib.pandas
prices.ts.SMA(20)
```

### `mintalib.polars` тАФ `ts` accessor
Registers a `ts` accessor on polars DataFrame, Series, and Expr.
```python
import mintalib.polars
prices.ts.SMA(20)
```

---

## The `wrap_*` Decorators (`mintalib.model`)

Three decorators bridge core functions to the higher-level interfaces:

- `wrap_function(calc_func)` тАФ creates a function wrapper with type conversion
- `wrap_indicator(calc_func)` тАФ creates a callable indicator class
- `wrap_expression(calc_func)` тАФ creates a polars expression factory

All decorators set `__module__`, `__name__`, `__doc__`, and `__signature__` on the wrapper so pdoc documents them correctly.

## Code Generation

`functions.py`, `indicators.py`, `expressions.py` are generated by Jupyter notebooks:

```
scripts/make-functions.ipynb    тЖТ src/mintalib/functions.py
scripts/make-indicators.ipynb   тЖТ src/mintalib/indicators.py
scripts/make-expressions.ipynb  тЖТ src/mintalib/expressions.py
```

Run `inv make` to recompile Cython and regenerate all derived files.

## `price` Helper

`calc_price(prices, item)` selects a price column by name. Supported values:

| item | alias | formula |
|---|---|---|
| `'close'` | (default) | close |
| `'open'` | | open |
| `'high'` | | high |
| `'low'` | | low |
| `'avg'` | `'ohlc4'` | (open + high + low + close) / 4 |
| `'mid'` | `'hl2'` | (high + low) / 2 |
| `'typ'` | `'hlc3'` | (high + low + close) / 3 |
| `'wcl'` | `'hlcc4'` | (high + low + 2├Чclose) / 4 |

---

## Key Design Decisions

- **`lag` not `shift`** тАФ `shift` was removed as it duplicates `lag` with a numpy-style name. `lag` is the standard TA name. `lag` raises on negative period; `shift` accepted negative (forward shift) but was deemed unnecessary.
- **Accessor opt-in** тАФ `mintalib.pandas` and `mintalib.polars` register the `ts` accessor at import time. They are NOT imported by `mintalib.__init__` тАФ users must explicitly import them.
- **`eval` supports both pandas and polars** тАФ uses `DataFrame.eval` for pandas and `DataFrame.sql` for polars.
- **`__pdoc__` to hide internals** тАФ `reflib`, `builder`, `testing` are hidden from pdoc via `__pdoc__ = {...: False}` in `__init__.py`.
- **Two doc sets** тАФ `docs/` holds markdown (AI/text, committed to repo); GitHub Pages serves pdoc HTML generated by CI on every push.
