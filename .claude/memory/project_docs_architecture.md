---
name: Dual docs architecture
description: Two separate doc systems — markdown in docs/ for AI/text, pdoc HTML via GitHub Actions for GitHub Pages
type: project
---

The project maintains two sets of documentation with different purposes:

- `docs/` — markdown files generated by `scripts/make_docs.py`, committed to the repo. Intended for AI/text consumption and readable directly on GitHub. Includes `_config.yml` for Jekyll theming as fallback.
- GitHub Pages HTML — generated by pdoc via `.github/workflows/pages.yml` on every push to main, deployed automatically to `https://furechan.github.io/mintalib/`. Never committed to the repo.

**Why:** Markdown is better for AI tools and plain readability; pdoc HTML is more attractive for human browsing. Generating HTML in CI avoids committing build artifacts.

**How to apply:** When updating docs, run `python scripts/make_docs.py` for the markdown side. The HTML side is handled automatically by CI on push.
