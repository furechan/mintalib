# mintalib

## Build & Test

Install in editable mode (compiles the Cython extension):
```console
uv sync
```

Run tests:
```console
pytest
```

Generate docs (do not edit `docs/` directly):
```console
invoke docs
```

After editing any `.pxi` file, re-run `uv sync` to recompile.

## Project Conventions

**Naming**
- Functions (`mintalib.functions`) ‚Äî lower case: `sma`, `ema`, `macd`
- Indicators (`mintalib.indicators`) and Expressions (`mintalib.expressions`) ‚Äî upper case: `SMA`, `EMA`, `MACD`

**Input parameters**
- `prices` ‚Äî pandas or polars DataFrame with columns `open`, `high`, `low`, `close`, `volume` (all lower case)
- `series` ‚Äî pandas/polars series or numpy array

**Multi-column outputs** (e.g. `MACD`, `BBANDS`) return named tuples from core functions, tuples of expressions in `mintalib.expressions`, and polars structs in the polars accessor.

## Architecture

- `src/mintalib/cython/` ‚Äî one `.pxi` file per indicator; compiled into `core.pyx` via `_all_core.pxi`
- `src/mintalib/core.pyx` ‚Äî Cython extension, exposes `calc_*` functions
- `src/mintalib/functions.py` ‚Äî thin Python wrappers around `core`, primary stable interface
- `src/mintalib/indicators.py` ‚Äî callable indicator objects, work with pandas and numpy
- `src/mintalib/expressions.py` ‚Äî polars expression factory functions, polars-native interface
- `src/mintalib/model/expression.py` ‚Äî `wrap_expression` decorator that bridges core functions to polars
- `src/mintalib/polars.py`, `src/mintalib/pandas.py` ‚Äî experimental `ts` accessors

**Stable interfaces:** `mintalib.functions`, `mintalib.expressions`, `mintalib.indicators`

**Experimental:** `mintalib.polars`, `mintalib.pandas` (`ts` accessors)

## Adding a New Indicator

1. Create `src/mintalib/cython/<name>.pxi` implementing `calc_<name>`
2. Add an `include "<name>.pxi"` line in `src/mintalib/cython/_all_core.pxi`
3. Run `uv sync` to recompile
4. Run the relevant codegen notebooks in `scripts/` to update `functions.py`, `indicators.py`, and/or `expressions.py`
5. Add tests in `tests/`

## Code Generation

Several source files are generated by notebooks in `scripts/` ‚Äî do not edit them directly:

- `src/mintalib/functions.py` ‚Äî generated by `scripts/make-functions.ipynb`
- `src/mintalib/indicators.py` ‚Äî generated by `scripts/make-indicators.ipynb`
- `src/mintalib/expressions.py` ‚Äî generated by `scripts/make-expressions.ipynb`
